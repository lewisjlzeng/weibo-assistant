## 1. 安装 Playwright 及系统依赖

OpenClaw 自带 `playwright-core`（npm 库），但它只是一个不含 CLI 的纯库。要使用 `playwright install` 命令下载浏览器，需要额外安装 Python 版 Playwright。同时 Chromium 运行依赖一系列系统级共享库，需要提前装好。

### 1.1 安装 Chromium 运行所需的系统依赖

```bash
sudo apt-get update
sudo apt-get install -y \
    libnss3 libatk1.0-0t64 libatk-bridge2.0-0t64 libcups2t64 \
    libxcomposite1 libxrandr2 libgbm1 libpango-1.0-0 \
    libpangocairo-1.0-0 libasound2t64 libatspi2.0-0t64 \
    libxdamage1 libxshmfence1
```

这些是 Headless Chromium 在 Linux 上运行的必要依赖。缺少任何一个都会导致浏览器启动失败并报 `shared library not found` 错误。

### 1.2 安装 Python 版 Playwright

```bash
pip3 install playwright
```

安装结果：
- 版本：Playwright 1.58.0
- 路径：`~/.local/lib/python3.12/site-packages/playwright/`
- CLI 命令：`/usr/local/bin/playwright`（安装后可直接使用 `playwright install chromium` 等命令）

> **说明**：OpenClaw 内部依赖的是 npm 包 `playwright-core@1.58.2`（随 OpenClaw 自动安装），负责在运行时通过 CDP 协议驱动浏览器。这里安装的 Python 版 Playwright 主要用于提供 `playwright install` CLI 命令来下载和管理浏览器二进制文件。两者版本需保持大版本一致（都是 1.58.x）。

## 2. 安装 Playwright Chromium 浏览器

Playwright 安装好后，正常情况下直接运行 `playwright install chromium` 即可自动下载浏览器。但服务器在国内，默认从海外 CDN 下载会卡住。改用国内镜像手动安装。

```bash
# 从 npmmirror 下载 Chromium 和 Headless Shell
wget -O /tmp/chrome-linux64.zip \
  'https://registry.npmmirror.com/-/binary/chrome-for-testing/145.0.7632.6/linux64/chrome-linux64.zip'
wget -O /tmp/chrome-headless-shell-linux64.zip \
  'https://registry.npmmirror.com/-/binary/chrome-for-testing/145.0.7632.6/linux64/chrome-headless-shell-linux64.zip'

# 解压到 Playwright 缓存目录（注意 headless_shell 用下划线）
mkdir -p ~/.cache/ms-playwright/chromium-1208
mkdir -p ~/.cache/ms-playwright/chromium_headless_shell-1208
unzip /tmp/chrome-linux64.zip -d ~/.cache/ms-playwright/chromium-1208/
unzip /tmp/chrome-headless-shell-linux64.zip -d ~/.cache/ms-playwright/chromium_headless_shell-1208/

# 写入安装标记（Playwright 靠这个文件判断是否已安装）
echo '1208' > ~/.cache/ms-playwright/chromium-1208/INSTALLATION_COMPLETE
echo '1208' > ~/.cache/ms-playwright/chromium_headless_shell-1208/INSTALLATION_COMPLETE

# 设置执行权限
chmod +x ~/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome
chmod +x ~/.cache/ms-playwright/chromium_headless_shell-1208/chrome-headless-shell-linux64/chrome-headless-shell

# 创建软链接（OpenClaw 只搜索 /usr/bin/chromium 等固定路径）
sudo ln -sf ~/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome /usr/bin/chromium
```

安装结果：
- 路径：`~/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome`
- 版本：Chrome for Testing 145.0.7632.6

## 3. 修改 OpenClaw 配置

改了 `~/.openclaw/openclaw.json` 中的两处配置：

### 3.1 启用浏览器（headless 模式）

```json
"browser": {
    "enabled": true,
    "headless": true,
    "noSandbox": true,
    "defaultProfile": "openclaw",
    "profiles": {
        "openclaw": {
            "cdpPort": 18800,
            "color": "#FF4500"
        }
    }
}
```

- `headless: true` — 无头模式，服务器没有图形界面必须开
- `noSandbox: true` — Linux 服务器必须关闭沙箱

### 3.2 工具配置改为 full

```bash
openclaw config set tools.profile full
```

将 `tools.profile` 从 `"coding"` 改为 `"full"`。只有 `full` 模式才包含 browser 系列工具，`coding` 模式下 Agent 无法操作浏览器。

### 3.3 重启生效

```bash
openclaw gateway restart
openclaw browser start
```

## 4. 创建微博助手 Skill

创建了两个文件：

### 4.1 SKILL.md（技能定义）

```
~/.openclaw/workspace/skills/weibo-assistant/SKILL.md
```

这是 OpenClaw 的 Skill 文件，定义了 Agent 在用户说"登录微博"、"微博消息"等关键词时应该怎么操作。核心内容包括：

- **登录流程**：导航到移动端登录页 → 截取二维码面板 → 发给用户 → 等扫码 → 保存 Cookie
- **信息抓取**：私信、评论、点赞、热搜、首页动态的页面地址和 API 端点
- **Cookie 管理**：每次任务前检查有效性，过期前提醒重新扫码
- **推送格式模板**：统一的消息排版格式

关键决策：**所有 URL 统一使用移动端 `m.weibo.cn`**，不用 PC 端 `weibo.com`。因为 PC 端会检测到服务器 IP（腾讯云数据中心）并拦截，移动端不会。

### 4.2 Cookie 管理脚本

```
~/.openclaw/workspace/skills/weibo-assistant/scripts/weibo_cookies.py
```

Python 脚本，提供四个命令：
- `check` — 检查 Cookie 是否有效（看 SUB 的 expires）
- `save` — 从 stdin 读取 JSON Cookie 数组保存到本地文件
- `load` — 输出已保存的 Cookie
- `export` — 导出为 `openclaw browser cookies set` 命令格式

Cookie 存储路径：`~/.openclaw/data/weibo/cookies.json`
