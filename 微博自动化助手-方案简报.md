# 微博自动化助手 — 实现方案简报

## 一、目标

在腾讯云服务器上部署 AI Agent（OpenClaw），通过企业微信与用户交互，自动完成微博登录、信息抓取和定时推送，**用户只需在企业微信中发消息即可获取微博动态摘要**。

## 二、整体架构

```
用户（企业微信） ⟷ OpenClaw Agent（腾讯云服务器） ⟷ 微博移动端（m.weibo.cn）
                        │
                        ├── Headless Chromium（浏览器自动化）
                        ├── Cookie 持久化存储
                        └── 定时任务引擎（cron）
```

核心技术选型：**OpenClaw + Headless Chromium + 微博移动端 API**。不依赖微博官方开放平台 API（已大幅收紧权限），而是通过浏览器自动化模拟真实用户访问移动端网页和接口。

## 三、核心实现步骤

### 第 1 步：服务器环境搭建

在腾讯云 Ubuntu 服务器上部署 OpenClaw Agent，安装 Headless Chromium 浏览器，配置企业微信消息通道。

| 组件 | 版本 / 配置 |
|------|------------|
| 服务器 | Ubuntu 24.04 LTS，腾讯云 |
| OpenClaw | 2026.3.13，Gateway 端口 18789 |
| 浏览器 | Chromium（Headless 模式，无图形界面） |
| AI 模型 | MiniMax-M2.5 |
| 消息通道 | 企业微信应用 + 企业微信机器人 |

### 第 2 步：微博登录（首次需扫码，后续自动免登）

**首次登录**：用户在企业微信中发送"登录微博" → Agent 打开微博移动端登录页 → 截取二维码发给用户 → 用户用微博 APP 扫码 → Agent 自动保存登录凭证（Cookie）到本地文件。

**后续登录**：Agent 从本地文件加载已保存的 Cookie，注入浏览器，直接恢复登录态，无需再次扫码。

> 关键决策：使用微博**移动端**（m.weibo.cn）而非 PC 端（weibo.com）。PC 端会检测到云服务器 IP 属于数据中心并拦截，移动端的反爬策略相对宽松，同一服务器 IP 可正常访问。

### 第 3 步：信息抓取

登录成功后，Agent 通过浏览器访问微博移动端页面和 API，抓取以下信息：

| 功能 | 数据来源 | 触发方式 |
|------|---------|---------|
| 私信 / 系统通知 | m.weibo.cn/api/msg/list | 用户指令 或 定时 |
| 评论 | m.weibo.cn/api/msg/cmt | 用户指令 或 定时 |
| 点赞 | m.weibo.cn/api/msg/attitude | 用户指令 或 定时 |
| 热搜榜 TOP15 | m.weibo.cn/api/container/getIndex | 用户指令 或 定时 |
| 首页关注动态 | m.weibo.cn/feed/friends | 用户指令 或 定时 |

### 第 4 步：定时推送

配置两个定时任务，自动抓取并通过企业微信推送微博动态摘要：

| 任务 | 执行时间 | 推送内容 |
|------|---------|---------|
| 早间推送 | 每天 09:00 | 热搜 TOP5 + 新消息/评论/点赞 + 首页热门动态 |
| 晚间推送 | 每天 20:00 | 同上 |

### 第 5 步：Cookie 生命周期管理

自动监控登录凭证的有效性，在过期前提醒用户重新扫码：

| Cookie | 有效期 | 说明 |
|--------|-------|------|
| SUB（核心凭证） | 365 天 | 判断登录态的关键 |
| ALF（自动登录标记） | 30 天 | 实际瓶颈，部分接口依赖 |
| WBPSESS（会话） | 1 天 | 页面级会话，自动刷新 |

实际效果：**一次扫码可维持至少 30 天的稳定使用**，到期前 Agent 会主动提醒重新扫码。

## 五、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 微博更新反爬策略 | 移动端接口可能被封 | 监控接口返回状态，及时调整 UA / 请求频率 |
| Cookie 提前失效 | 定时推送中断 | 每次任务前自动检查 Cookie 有效性，失效时立即通知 |
| 服务器 IP 被封 | 所有请求被拦截 | 控制请求频率，避免高频访问；必要时更换 IP |
| 微博页面结构变更 | DOM 选择器失效 | Skill 中使用多种备选选择器，定期验证 |
